### 8. Автоматическое обнаружение устройств по Heartbeat
- **Суть:** Пульт может получать heartbeat-пакеты от других устройств, которых ещё нет в списке избранного. При получении такого пакета на экране появляется уведомление с предложением добавить устройство в избранное.
- **Что нужно сделать:**
  - Определить формат heartbeat-пакета (например, `struct Heartbeat { uint8_t version; char name[32]; }`).
  - В `Control` добавить обработчик `onReceiveFromUnknown`, который при получении пакета от неизвестного MAC проверяет, является ли он heartbeat (например, по сигнатуре). Если да — временно сохранить MAC.
  - В UI добавить всплывающее окно/уведомление: "New device XX:XX:XX:XX:XX:XX. Add to favorites? [Yes/No]".
  - При подтверждении — добавить MAC в `favorites`, сохранить конфиг, обновить `PeersPage`.

### 9. CLI (Command Line Interface) для отладки и настройки
- **Суть:** Интерфейс командной строки через последовательный порт для диагностики и изменения параметров устройства без перепрошивки.
- **Что нужно сделать:**
  - Выбрать и интегрировать легковесную библиотеку CLI. Варианты:
    - [ESP-Shell](https://github.com/vvb333007/espshell) — простая интеграция, небольшой оверхед, поддерживает автодополнение, историю, паузу выполнения скетча.
    - [ESP32 WiFi CLI](https://github.com/hpsaturn/esp32-wifi-cli) — удобна для управления сетью, но требует больше ресурсов.
    - `esp_console_repl` из ESP-IDF — самый мощный, но сложный для интеграции с Arduino.
  - **Рекомендация:** Начать с `ESP-Shell`. Добавить команды:
    - `peers` — показать список избранных пиров.
    - `add <MAC>` — добавить пир.
    - `del <MAC>` — удалить пир.
    - `active <MAC>` — выбрать активный пир.
    - `mode [raw|mavlink]` — переключить режим управления.
    - `reboot` — перезагрузить устройство.
    - `save` — сохранить конфиг.
    - `reset` — сбросить конфиг к заводским.
    - `info` — показать текущий статус (MAC, режим, активный пир, уровень заряда батареи и т.д.).
  - Реализовать обработчик команд, вызывающий соответствующие методы `ConfigManager` и `Control`.
